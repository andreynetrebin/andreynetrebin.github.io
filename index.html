Обо мне

Моя цель занять позицию дата-инженера, чтобы применять свои навыки в обработке и анализе данных для создания эффективных и масштабируемых решений.

У меня есть опыт работы в различных областях обработки и анализа данных. Мои ключевые навыки включают:

  • Языки программирования: Python (скрипты, десктопные приложения с PyQT, веб-приложения с Flask и Django).
  • Работа с базами данных: Опыт работы с Greenplum для аналитической обработки данных и выполнения сложных SQL-запросов, включая использование PXF для интеграции с внешними источниками, а также с другими SQL и NoSQL базами данных, такими как DB2, Netezza, PostgreSQL, MySQL, ClickHouse и MongoDB.
  • Обработка данных: Использование Pandas и PySpark для работы с данными из различных источников (базы данных, API, JSON, CSV, XML).
  • Работа с HDFS: Опыт работы с Hadoop Distributed File System для хранения и обработки больших объемов данных.
  • Визуализация данных: Создание интерактивных дашбордов с использованием Dash и Grafana.
  • Автоматизация задач: Использование Selenium для автоматизации взаимодействия с веб-приложениями.
  • Инструменты и технологии: Опыт работы с брокерами очередей (IBM MQ, Kafka), стеком ELK, Apache Airflow, Docker и Camunda.
  • Системы контроля версий: Уверенное использование Git для управления версиями кода, совместной работы над проектами и обеспечения надежности разработки.

Работа с базами данных. Разработка ETL. Витрины.

  • Универсальный коннектор к базам данных - Проект представляет собой библиотеку для подключения к различным базам данных через JDBC, реализующую потокобезопасный Singleton на Python. - Она поддерживает различные СУБД, и позволяет выполнять SQL-запросы с возвратом результатов в формате DataFrame Pandas. - Библиотека обеспечивает надежную обработку соединений с автоматическим восстановлением и детальным логированием, что упрощает интеграцию и анализ данных.
  • ETL-процесс для витрины данных на PostgreSQL - Проект создает витрину данных для минимизации нагрузки на продуктовый сервер, обеспечивая надежный доступ к важной информации, используемой для анализа, аудита бизнес-процессов и устранения инцидентов.
  • Специфика работы с Netezza - Проект посвящен работе с аналитической платформой Netezza, направленный на оптимизацию хранения и обработки больших объемов данных. - В проекте рассматриваются методы выполнения стандартных SQL-запросов, использование встроенных инструментов мониторинга, таких как команда EXPLAIN и системные представления, а также методы оптимизации запросов. - Основные выводы подчеркивают важность правильной архитектуры, мониторинга и регулярного анализа распределения данных для достижения высокой производительности в аналитических задачах.
  • Специфика работы с Greenplum - Проект посвящен оптимизации хранения и обработки данных в распределенной системе управления базами данных Greenplum. - Рассматриваются архитектура MPP, виды таблиц, методы сжатия и распределения данных, а также встроенные инструменты мониторинга и методы оптимизации запросов. - Основные аспекты включают использование хеширования и реплицированного распределения для эффективного распределения данных, применение PXF для интеграции данных из различных источников, а также анализ планов выполнения запросов для выявления узких мест. - Выводы подчеркивают важность правильной организации хранения данных и мониторинга для достижения высокой производительности в аналитических задачах.

Работа с API

  • REST API Tool - Проект представляет собой библиотеку на Python для работы с REST API, которая упрощает интеграцию между системами и повышает эффективность. - Она поддерживает все HTTP методы, аутентификацию (Bearer, Basic, API ключи), операции с файлами и предоставляет детализированное логирование.
  • WebHDFS Клиент - Проект представляет собой клиент WebHDFS для Python, который использует пользовательский инструмент API и поддерживает все основные операции с HDFS, включая создание директорий, загрузку и чтение файлов, а также удаление и переименование. - Клиент также предоставляет возможность получения метаданных файлов и директорий, аутентификацию с использованием токенов, логирование запросов и ответов для удобства отладки, а также обработку ошибок с подробными сообщениями.

Прочее

  • Переотправки - Проект представляет собой Flask-приложение для автоматического формирования и отправки сообщений (XML/JSON) в IBM MQ, предназначенное для оперативного устранения инцидентов в технологических и бизнес-процессах.
  • Мониторинг - Проект представляет собой приложение для мониторинга обработки данных в технологических процессах, которое включает рендеринг дашбордов и отправку уведомлений. - Приложение разработано с использованием Python и Dash, что позволяет эффективно визуализировать данные и управлять процессами.
  • RPA Скрипт - Проект представляет собой RPA скрипт, который автоматизирует процесс извлечения данных о рабочих заданиях (заявках) из веб-интерфейса IBM SmartCloud Control Desk, их принятия в работу и сохранения в Excel-файлы для дальнейшей обработки или отчетности. - Этот инструмент значительно упрощает управление заявками, повышая эффективность работы с технологическими процессами.

Проекты по курсу"Data Engineer с нуля до junior"

  • Разработка ETL-пайплайна c PySpark и ClickHouse - Разработка ETL-пайплайна для обработки данных из CSV файла с использованием PySpark и ClickHouse, управляемого Apache Airflow и развернутого в Docker-контейнерах через Docker Compose.
  • Разработка ETL-пайплайна c PySpark и PostgreSQL - Этот проект предназначен для обработки данных о полетах с использованием Apache Airflow и PySpark. - Он загружает данные из CSV файла, обрабатывает их, сохраняет в базу данных PostgreSQL и выполняет анализ.
  • Разработка ETL-пайплайна c PySpark, PostgreSQL, ClickHouse и Grafana - Этот проект представляет собой конвейер обработки данных, который включает в себя генерацию, проверку, очистку, загрузку и анализ данных о продажах. - Данные извлекаются из PostgreSQL, обрабатываются с помощью Apache Spark и затем загружаются в ClickHouse для дальнейшего анализа. - Также в проекте создан дашборд в Grafana для визуализации данных.
